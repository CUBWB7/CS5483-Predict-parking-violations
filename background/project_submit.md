# Group report

What to submit?
The report should be approximately 15-pages, excluding source code and references.
Source code should be submitted along with the report.
Although you may submit your report/code in a zip file, you are encouraged to use Notebooks and Git repository. If you put your report/code in the group server or a public Git repo, you can simply submit a link to them.
The report should use the style for a research report. 
it should have an abstract that summarizes the report, an introduction that motivates the problem, and a conclusion that summarizes the results. See the section on Research Report in The Mayfield Handbook of Technical & Scientific WritingLinks to an external site..
There should be adequate in-text citations, e.g., source of the real-world data, related approach and results. See the section on Citing Sources in The Mayfield HandbookLinks to an external site..
Note also that you can present jupyter notebook as slides with the jupyterlab-deckLinks to an external site. extension which is already available on JupyterHub, in which case you do not need to create/submit Slides separately. In addition, mystmdLinks to an external site. may be used to compile jupyter notebooks into a report.

Rubrics:

Report score (3 points)
Is the problem described clearly with proper citations and other essential elements Links to an external site.for a technical report?
Are there sufficient details for the results to be reproducible by others?
Correctness score (3 points)
Does the project contain the required elements?
Are the results correct?
Are the learning process and evaluation methods appropriate, leading to correct results?
Technical element score (3 points)
How sophisticated are the techniques used for preprocessing, learning, and evaluation?
Is the quality of the mined knowledge better than existing ones?
Are there meaningful generalizations to related classes of problems?
Team spirit (3 points)
Can the team challenge other teams successfully for the group presentation?
Can the team members work together efficiently?
Is the workload evenly divided among members?
