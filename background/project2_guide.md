# CS5483 Project 2 — 项目要求分析与指导

> 本文档是对 Project 2 要求的中文梳理，帮助小组成员快速理解项目全貌。提交物需用英文。

---

## 一、项目总览

这是一个**小组项目**，核心任务是：**从一个真实世界的数据集中，通过数据挖掘的方法提取有价值的知识**。

- 项目总分 **15 分**，占课程总评的 **15%**。
- 需要提交：展示视频 + Peer Review + 书面报告。

---

## 二、项目必须包含的四个核心环节

| 环节 | 说明 |
|------|------|
| **1. Objective（目标）** | 明确你要从数据中挖掘什么知识、解决什么问题 |
| **2. Data Preprocessing（数据预处理）** | 对原始数据进行清洗、转换、特征工程等，使其适合挖掘 |
| **3. Learning Algorithm（学习算法）** | 使用机器学习/数据挖掘算法从数据中提取知识 |
| **4. Evaluation（评估）** | 对挖掘出的知识/模型进行科学评估 |

这四个环节缺一不可，评分时会检查是否包含。

---

## 三、需要提交的内容

### 1. 小组展示视频（Presentation Video）
- **时长**：15 分钟
- 提交到 Canvas 上的 Group Presentation 作业
- 会被其他同学 **peer review**（同学互评）
- 需通过 Panopto 上传

### 2. 同行评审（Peer Review）
- 每位同学需要评审 **3 个其他小组** 的展示视频

### 3. 小组报告（Report）
- 大约 **15 页**（不含源代码和参考文献）
- 源代码需随报告一起提交
- 可以提交 zip 文件，也可以用 **Notebook + Git 仓库**（推荐）
- 如果代码放在 group server 或公开 Git repo 上，提交链接即可

**报告格式要求**（研究报告风格）：
- **Abstract**（摘要）：概括全文
- **Introduction**（引言）：阐述问题的动机和背景
- **Conclusion**（结论）：总结结果
- 需要有充分的 **in-text citations**（文内引用），如数据来源、相关方法和结果

## 四、评分标准详解（5 个维度，每项 0-3 分）

| 维度 | 关键点 | 拿高分要点 |
|------|--------|-----------|
| **Presentation（3分）** | 问题阐述是否清晰？可视化是否有效？能否回答观众提问？ | 用图表说话，控制时间，准备 Q&A |
| **Report（3分）** | 报告是否包含必要元素？描述是否清楚？代码是否可复现？ | 格式规范，引用充分，代码可跑 |
| **Correctness（3分）** | 是否包含四个核心环节？结果是否正确？方法是否合理？ | 四个环节完整，评估方法得当 |
| **Technical Elements（3分）** | 技术手段是否有一定深度？结果是否优于已有方法？ | 用较先进的技术，对比 baseline |
| **Team Spirit（3分）** | 团队协作是否高效？分工是否均匀？ | 明确分工，互评时积极提问 |

## 五、选题策略

### 选题原则
1. **使用真实世界数据集**（不能是玩具数据）
2. **不能重复使用以前做过的项目**（避免自我抄袭嫌疑）
3. **目标要明确**：不是"分析数据"，而是"预测 XX"或"发现 XX 的模式"

### 推荐数据来源（可靠性高）
- **UNICEF Data** (data.unicef.org) — 儿童福利相关数据
- **World Bank Open Data** (data.worldbank.org) — 全球发展数据
- **Data.gov.hk** (data.gov.hk) — 香港开放政府数据
- **Eurostat** (ec.europa.eu/eurostat) — 欧洲统计数据
- **Challenge Data** (challengedata.ens.fr) — 数据挖掘竞赛数据
- **Google Dataset Search** (datasetsearch.research.google.com) — 数据集搜索引擎

### 需要谨慎使用的来源
- **UCI、Kaggle、OpenML** — 数据集丰富但很多已被大量研究，或是合成数据。使用时需要：
  - 验证数据的真实性，引用原始来源
  - 明确说明你的方法和结果与已有工作的区别

### 好选题的特征
- **有现实意义**的问题（如预测空气质量、检测欺诈交易、疾病预测等）
- **数据量适中**（太小没法挖掘，太大处理不了）
- **有足够的特征**可以做预处理和特征工程
- **有明确的评估指标**（如准确率、F1-score、AUC 等）

## 六、建议的项目开展流程

### 第 1 阶段：选题与数据探索
1. 小组讨论，确定 2-3 个候选方向
2. 分别去找对应的数据集，评估数据质量（大小、缺失值、特征数量）
3. 确定最终选题，明确挖掘目标

### 第 2 阶段：数据预处理
- 处理缺失值（填充或删除）
- 处理异常值
- 特征选择 / 特征工程
- 数据标准化 / 归一化
- 如果是文本数据，还需要分词、向量化等

### 第 3 阶段：建模与实验
- 选择合适的学习算法（分类、回归、聚类等，取决于目标）
- **尝试多种算法**进行对比
- 调参优化
- 使用交叉验证等方法确保结果可靠

### 第 4 阶段：评估与分析
- 使用合适的评估指标
- 分析结果，解释模型发现的知识
- 与已有方法 / baseline 进行对比

### 第 5 阶段：撰写报告 + 录制视频
- 报告按研究论文格式撰写（Abstract → Introduction → Method → Results → Conclusion）
- 视频控制在 15 分钟以内，确保可视化清晰

## 七、重要注意事项

1. **必须用编程完成**（不能用 WEKA 图形界面），可用 Python 的 `python-weka-wrapper3`、scikit-learn、PyTorch 等
2. **代码要可复现** — 报告评分的重要标准，确保别人拿到代码能跑出相同结果
3. **引用要规范** — 数据来源、参考论文和方法都要正确引用
4. **避免抄袭** — 特别是用 Kaggle 数据集时，不能照搬别人的 Notebook
5. **分工要均匀** — Team Spirit 会考察每个成员的贡献
6. **注意截止日期** — 视频和报告的 deadline 可能不同，去 Canvas 确认
7. **Technical Elements 想拿高分**需要：使用较先进的技术 + 结果优于现有方法 + 对相关问题有泛化讨论
