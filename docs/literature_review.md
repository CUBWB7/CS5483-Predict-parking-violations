## Part A: 直接相关论文 — 深度学习停车违章预测

### Paper 1: Deep Learning for On-Street Parking Violation Prediction

- **作者**：Vo Thien Nhan（胡志明市理工大学）
- **来源**：arXiv:2505.06818, 2025
- **文件**：`deep_learning_parking_violation_2025.pdf`

#### 问题与数据

本文预测城市各停车区域的**违规率**（违章停车数 / 区域容量），与我们的 ChallengeData #163 挑战**问题设定完全一致**。

- **数据来源**：THESi 系统（希腊塞萨洛尼基的路边公共停车系统）
- **规模**：约 4,700 个停车位，分布在 396 个区域，380 万次扫描记录
- **时间范围**：7:00-19:00（执法时段，与我们数据的 6:00-19:00 一致）
- **平均违规率**：0.41（与我们数据的均值 ~0.50 接近）

> **重要发现**：本文使用的 THESi 系统数据很可能与 ChallengeData #163 来自同一来源。

#### 模型架构

6 层残差神经网络 + 1 个输出层：

```
输入(28维) → 512 → 256 → 128 → 64 → 128 → 32 → 1(输出)
                          ↓              ↑
                        残差跳跃连接（bottleneck）
```

- 隐藏层激活：ReLU
- 输出层激活：Sigmoid（确保输出在 [0,1]）
- 优化器：Adamax，初始学习率 0.001
- 学习率调度：指数衰减，衰减率 0.25
- **目标值重归一化**：将 [0,1] 缩放到 [0.1, 0.9]，避免 Sigmoid 饱和

#### 特征工程（28 维特征）

| 类别 | 特征 | 维度 | 说明 |
|------|------|------|------|
| 地理编码 | 到 19 个兴趣点(PoI)的距离 | 19 | 比原始 GPS 坐标更好地捕捉空间语义 |
| 区域属性 | 区域容量（停车位数） | 1 | 对应我们数据中的 `total_count` |
| 时间特征 | 星期、日、月、时段 | 4 | 使用 **sine 编码**处理周期性 |
| 天气特征 | 温度、湿度 | 2 | **6 小时窗口平均**（当前+前 5 小时） |
| 指示特征 | 公共假日、疫情期间 | 2 | 二元标志 |

**时间特征的 sine 编码**（关键技巧）：
- 星期：`sin(2π × w/7)`，其中 w ∈ {0,1,...,6}
- 月份：`sin(2π × (m-1)/12)`，其中 m ∈ {1,...,12}
- 这样编码使得 **周一和周日是"相邻的"**，而不是数值上差距最大的

**天气特征的 6 小时窗口**：
- `x_T = (1/6) × Σ(T_{t-i})`，取当前小时和前 5 小时的平均值
- 原因：天气对交通行为的影响有**延迟效应**

#### 核心创新：高斯平滑

这是论文最重要的贡献。问题在于：警察扫描是**稀疏的**（不是每个区域每个小时都有记录），导致标签**不连续且有噪声**。

**解决方案**：用高斯函数将观测到的违规率分配到相邻时段：

```
y = (1/|S|) × Σ exp(-d_s/σ) × p_s
```

其中 σ = 210 分钟，d_s 是到当前时段中心的分钟距离。

**效果**：MAE 从 0.175 降到 0.146（改善 16.5%），总共比基线（均值预测器 0.251）改善 41.8%。

#### 对我们项目的启示

1. **PoI 距离特征**比原始 GPS 坐标更有效 — 我们可以尝试类似的空间语义编码
2. **Sine 编码**处理周期性特征是标准做法
3. **6 小时天气窗口**比单一时刻的天气值更有信息量
4. **高斯平滑**处理稀疏标注 — 如果我们的数据也有类似问题，可以借鉴
5. 残差网络架构相对简单，可作为深度学习方案的参考

### Paper 2: Predicting On-Street Parking Violation Rate Using Deep Residual Neural Networks

- **作者**：Karantaglis, Passalis, Tefas（亚里士多德大学）
- **来源**：Pattern Recognition Letters, Vol.163, 2022, pp.82-91
- **文件**：`1-s2.0-S0167865522002938-main.pdf`

#### 与 Paper 1 的关系

Paper 2 是 Paper 1 的**基础论文**，两者使用相同的数据集、架构和方法论，但 Paper 2 提供了更详尽的**消融实验和对比分析**。

#### 消融实验：特征重要性（最有价值的发现）

论文逐步添加特征组，测量 MAE 变化：

| 使用的特征 | MAE | 相对改善 |
|------------|-----|---------|
| 仅 PoI 距离 | 0.211 | 基准 |
| + 区域容量 | 0.206 | 2.4% |
| **+ 时间特征** | **0.185** | **10.2%（最大跳跃）** |
| + 天气特征 | 0.176 | 4.9% |
| + 假日指标 | 0.173 | 1.7% |
| + 疫情指标 | 0.169 | 2.3% |

**关键结论**：**时间特征贡献最大**（10.2%），其次是天气（4.9%），假日/疫情最小。统计检验确认差异显著（p=0.002）。

#### 与先前工作对比

| 方法 | MAE | MSE |
|------|-----|-----|
| Gao et al. (2019) | 0.191 | 0.0561 |
| **本文方法** | **0.169** | **0.0497** |
| 改善 | 11.5% | 11.4% |

#### 其他关键发现

- **违规率时间分布**：午间 13:00-14:00 最高（~0.50）
- **空间分布**：上城区和偏远区域违规率更高
- **COVID-19 影响**：疫情期间平均违规率从 0.39 上升到 0.46
- **扫描覆盖率**：每次检查约覆盖 76% 的区域

#### 对我们项目的启示

1. **特征重要性排序**明确指导我们的特征工程优先级：时间 > 天气 > 指标变量
2. 该方法比 2019 年的先前工作改善 11%+，证明深度学习在此问题上的有效性
3. 消融实验的做法值得在我们的报告中复现（逐步添加特征组，对比 Spearman）

### Paper 3: Predicting the Spatiotemporal Legality of On-Street Parking Using Open Data and ML

- **作者**：Gao, Li, Liang, Marks, Kang, Li
- **来源**：Annals of GIS, 2019
- **文件**：`Predicting the spatiotemporal legality of on-street parking using open data and machine learning.pdf`

#### 问题与数据

- **研究区域**：纽约市（曼哈顿、皇后区、布鲁克林、布朗克斯、史泰登岛）
- **数据量**：1080 万张停车罚单（2017-2018 财年）
- **核心创新**：在 4 个空间尺度上比较模型表现（点级、街道级、人口普查区级、1km 网格级）

#### 方法论

测试了 6 种 ML 模型：线性回归、SVM、决策树、**Random Forest**、GBRT、DNN。

#### 特征工程

- **POI 特征**：SafeGraph 提供的 137,000+ 个兴趣点，分 23 个 NAICS 类别
  - 零售贸易、医疗保健、餐饮服务是与违章最相关的 POI 类别
- **人类流动性**：智能手机用户的访问模式（独立访客数、时间分布）
- **街道特征**：宽度、长度、类型、是否单行道
- **时间特征**：168 个小时时段（7天 × 24小时）

#### 关键发现

1. **Random Forest 在所有空间尺度上表现最好**（RMSE 最低，准确率 0.82-0.88）
2. **空间尺度影响显著**：更细的空间分辨率（点级）比粗糙尺度（网格级）表现更好
3. **时间自相关**：3 小时滞后的自相关系数 0.28-0.55，暗示**滞后特征**很重要
4. **不平衡数据处理**：重采样后分类性能显著提升

#### 对我们项目的启示

1. **RF 作为强基线**：确认 Random Forest 的有效性，与挑战的 baseline 模型一致
2. **POI 密度特征**：按类别的 POI 密度是重要预测因子（零售、餐饮、医疗）
3. **多尺度分析**：空间粒度影响预测精度，我们需要找到最佳的网格大小
4. **时间滞后特征**：3 小时的时间自相关暗示可以构建滞后特征
5. **不平衡处理**：我们的数据中 27% 的 invalid_ratio=1.0 和 16% 的 invalid_ratio=0.0，可能需要特殊处理

### Paper 4: Short-term Parking Violations Demand Dynamic Prediction

- **作者**：Liu & Chen
- **来源**：Transportation (Springer), 2025
- **文件**：`s11116-025-10632-7.pdf`

#### 问题与数据

- **研究区域**：中国重庆市大渡口区
- **数据量**：80,572 条违章记录（2023年1-5月），233 个电子监控设备
- **核心创新**：MGTWM（多尺度地理时间加权回归）+ GAT + ALSTM 的混合架构

#### 三分支深度学习架构

```
分支1: GAT（图注意力网络）→ ResNet → aLSTM（注意力LSTM）
分支2: MGTWM层 → GAT → ResNet → aLSTM
分支3: 自然环境特征处理
→ 加权融合 → 预测输出
```

#### 特征工程

这篇论文的特征工程最为全面：

- **建成环境（AOI）**：住宅区、商业区、购物区、休闲区、医院、学校的面积
- **街道特征（POI）**：公司、休闲、餐饮、购物、服务类 POI 数量
- **交通供给**：到路边停车位、停车场、公交线路的距离
- **道路设计**：车道数、中央隔离带、人行护栏
- **自然环境**：天气（雨天/阴天）、温度、能见度
- **历史特征**：**同地点上周/昨天的违章数**（滞后特征）
- **空间范围**：每个设备 400m 半径的汇水区

#### 关键发现

1. **犯罪率、道路密度、人口密度**是最主要的驱动因素
2. **工作日违章多于周末**，高峰在 08:00-09:00 和 12:00-13:00（通勤和午休）
3. **季节差异**：冬季模式不同于夏季
4. **建成环境影响存在时空异质性**：同一因素在不同地方、不同时间的影响程度不同
5. 教育设施附近违章更多

#### 对我们项目的启示

1. **历史滞后特征**：如果能从数据中推断同一地点的历史违规率，这将是非常强的特征
2. **三分支架构**思路可借鉴：分别处理时间序列、空间关系、环境因素
3. **图注意力网络(GAT)**可建模区域间的空间关联
4. **工作日高峰模式**（08-09, 12-13）与我们数据中的 hour 分布一致
5. **通勤/午休分箱**：可据此设计时段分箱特征

### Paper 5: Spatio-temporal Heterogeneity in Street Illegal Parking — NYC Case Study

- **作者**：Sui, Feng, Zhang
- **来源**：Journal of Transport Geography, 2025
- **文件**：`1-s2.0-S096669232500153X-main.pdf`

#### 问题与数据

- **研究区域**：纽约市（重点曼哈顿），500m × 500m 网格（1,836 个格子）
- **数据量**：18,736,534 条违章记录（2023 年 1 月和 6 月）
- **方法**：贝叶斯层次模型（BYM + 零膨胀泊松 + INLA）

#### 核心方法论

采用 **Besag-York-Mollié (BYM) 模型**结合**零膨胀泊松分布 (ZIP)**：

- **BYM 模型**：η_i = μ + xβ + θ + φ
  - θ：结构化空间效应（通过 ICAR 捕捉空间自相关）
  - φ：非结构化空间效应（未测量的局部因素）
- **零膨胀泊松**：处理大量零值（低违章区域）
- **INLA**：高效贝叶斯推断

#### 特征体系

| 类别 | 特征 | 描述 |
|------|------|------|
| 社会经济 | 人口密度 (PD) | 0-80.94 千人/km² |
| 社会经济 | 犯罪率 (CR) | 0-373 起/km² |
| 交通 | 道路密度 (RD) | 0.61-275.18 km/km² |
| 交通 | 停车费 (PF) | $0-$4.59/小时 |
| 交通 | 出租车需求 (TD) | 代理停车需求 |
| POI | 住宅/政府/交通/教育/休闲/信息 | 各类 POI 密度 |
| 天气 | 温度/风速/湿度/气压/能见度/天气状况 | 多维度天气 |

#### 关键发现

1. **湿度(HM)是最显著的天气因子**（95% CI [0.015, 0.021]），比温度更重要
2. **SHAP 分析揭示非线性交互**：犯罪率×人口密度、温度×能见度有复杂交互效应
3. **零膨胀分布很关键**：低违章区域有大量零值，传统模型会低估
4. **空间自相关**：相邻区域的违章模式相似（ICAR 模型捕捉）
5. **不确定性量化**：外围区域的预测不确定性更高
6. **停车费的抑制效应**：高停车费区域违章更少

#### 对我们项目的启示

1. **零膨胀问题**：我们的数据 16% 为 0、27% 为 1 — 可能需要特殊处理（如分段建模）
2. **SHAP 可解释性分析**：项目报告中应使用 SHAP 展示特征重要性和交互效应
3. **湿度比温度更重要**：虽然我们的数据没有湿度，但降水可能起类似作用
4. **500m 网格**是合理的空间单位 — 可参考设计我们的空间离散化粒度
5. **空间自相关建模**：如果能获取或推断区域邻接关系，可增强模型

## Part C: 模型论文 — GBDT 算法

### Paper 6: LightGBM — A Highly Efficient Gradient Boosting Decision Tree

- **作者**：Ke, Meng, Finley, Wang, Chen, Ma, Ye, Liu（Microsoft Research）
- **来源**：NeurIPS 2017
- **文件**：`lightgbm_ke_2017.pdf`

#### 核心创新

解决了传统 GBDT 在大数据上的效率问题，提出两项关键技术：

**1. GOSS（基于梯度的单侧采样）**

原理：梯度大的样本对信息增益贡献大（训练不充分的样本），梯度小的样本已经学好了。

做法：
- 保留梯度最大的 a% 样本（如 top 10%）
- 从剩余 (1-a)% 中随机采样 b%（如 10%）
- 对小梯度样本乘以补偿权重 (1-a)/b

效果：仅用 10-20% 的数据就能获得接近全数据的信息增益估计，速度提升 ~2x。

**2. EFB（互斥特征捆绑）**

原理：高维稀疏数据中，很多特征互斥（同时为非零的情况极少），可以合并。

做法：
- 构建冲突图，贪心分组互斥特征
- 通过偏移合并特征值域

效果：稀疏特征上可获得高达 10x 加速。

**3. Leaf-wise 树生长**（vs XGBoost 的 Level-wise）

- LightGBM 选择增益最大的叶子分裂（Best-first）
- 产生更深、更不对称的树
- 相同叶子数量下更准确，但需要 `max_depth` 防止过拟合

#### 对我们项目的意义

LightGBM 是我们 **600 万行数据的首选模型**，原因：
- GOSS 在大数据上的采样策略正好匹配我们的场景
- 训练速度比 XGBoost 快 6-20x
- 内置正则化（GOSS + EFB）降低过拟合风险

### Paper 7: XGBoost — A Scalable Tree Boosting System

- **作者**：Chen & Guestrin（华盛顿大学）
- **来源**：KDD 2016
- **文件**：`xgboost_chen_2016.pdf`

#### 核心创新

**1. 正则化学习目标**

```
L(φ) = Σ l(ŷᵢ, yᵢ) + Σ Ω(fₖ)
其中 Ω(f) = γT + ½λ||w||²
```

显式的正则项（叶子数 T 的惩罚 + 叶子权重 w 的 L2 惩罚）防止过拟合。

**2. 二阶泰勒展开**

使用梯度 gᵢ 和 Hessian hᵢ 来优化目标函数，比仅用一阶梯度更精确。

**3. 加权分位数草图**

用于近似分裂点搜索，处理加权实例，时间复杂度从 O(d×n×log n) 降到 O(d×n×log buckets)。

**4. 稀疏感知分裂**

自动处理缺失值：为每个节点学习最优的"默认方向"（缺失值走左还是走右）。

#### LightGBM vs XGBoost 对比

| 维度 | LightGBM | XGBoost |
|------|----------|---------|
| 分裂策略 | 直方图（默认） | 精确贪心 + 直方图可选 |
| 数据采样 | GOSS（梯度感知） | 通用子采样 |
| 特征处理 | EFB 自动捆绑 | 无 |
| 树生长 | Leaf-wise（Best-first） | Level-wise |
| 速度 | 快 6-20x | 基准 |
| 正则化 | 简单形式 + GOSS/EFB 隐式正则 | 显式 L1+L2+γ |
| 缺失值 | 支持 | 默认方向学习 |
| 最适合 | >100 万行，高维稀疏 | 中等数据，需要最高精度 |

#### 推荐的超参数配置

**LightGBM 起步配置（我们的项目）**：

```python
params = {
    'objective': 'regression',
    'metric': 'rmse',            # 训练监控 RMSE
    'boosting_type': 'gbdt',
    'num_leaves': 31,            # 从默认值开始
    'learning_rate': 0.1,
    'num_iterations': 1000,
    'early_stopping_rounds': 50, # 用 Spearman 做早停！
    'lambda_l2': 1.0,
    'min_data_in_leaf': 20,
    'feature_fraction': 0.8,
    'bagging_fraction': 0.8,
    'bagging_freq': 5,
    'num_threads': -1,
}
```

**调参优先级**：
1. `num_leaves`（10-150）：控制模型复杂度
2. `learning_rate`（0.01-0.2）：控制收敛速度
3. `min_data_in_leaf`（10-50）：防止叶子过拟合
4. `feature_fraction`（0.7-1.0）：特征子采样
5. `lambda_l2`（0-10）：L2 正则化

**针对 Spearman 相关系数的优化策略**：
- Spearman 只看**排序**不看绝对值 → 模型只需正确排序预测值
- 用 Spearman（而非 RMSE）做验证集的早停指标
- 可尝试 Huber 损失（对异常值更鲁棒，减少极端值对排序的干扰）
- 考虑 LGBMRanker 直接优化排序

## 综合指导建议：从 7 篇论文中提取的可直接应用策略

### 1. 特征工程策略（按优先级排序）

| 优先级 | 策略 | 来源论文 | 应用方式 |
|--------|------|---------|---------|
| **最高** | 空间离散化 → 区域 ID → Target Encoding | Paper 1,2,3,5 | 将 GPS 网格化为区域，计算各区域历史违规率 |
| **最高** | Sine 编码处理周期性时间特征 | Paper 1,2 | `sin(2π×hour/24)`, `sin(2π×dow/7)`, `sin(2π×month/12)` |
| **高** | 6 小时天气窗口平均 | Paper 1,2 | 如果能获取历史天气序列，取当前+前 5 小时平均 |
| **高** | total_count 分箱 + 交互 | 数据分析 | 低 count 样本噪声大，按分箱做不同处理 |

### 2. 建模策略

### 4. 需要注意的风险

| 风险 | 来源 | 应对 |
|------|------|------|
| Target Encoding 数据泄露 | 用训练集统计编码测试集 | 使用 K-fold Target Encoding（每个 fold 只用其他 fold 的统计） |
| 小 total_count 样本噪声 | invalid_ratio 在 count=1 时只能为 0 或 1 | 参考 Paper 1 的 baseline 做法：过滤 count≥45 的记录，或对小 count 样本降权 |
| 600 万行内存压力 | 数据量大 | LightGBM 直方图算法天然优势；pandas 用 dtype 优化 |
| Spearman 不可直接优化 | 非可微指标 | 用 MSE/Huber 训练，Spearman 做验证早停 |
