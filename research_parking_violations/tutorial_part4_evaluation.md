## 一、模型评估 — 你的模型到底有多好？

### 1.1 为什么需要评估

训练完模型后，我们不能只看训练数据上的表现就下结论。这就像学生只在练习题上做对了不算数，得看考试（验证集/测试集）上的成绩才能反映真实水平。

模型评估的目的有两个：
- **判断模型是否"学到了真知识"**：而不是只是"背答案"（过拟合）
- **比较不同模型的优劣**：帮助我们选择最好的方案

### 1.2 主指标：Spearman 相关系数

这是 ChallengeData #163 的**唯一官方评分标准**，在 Part 1 中我们已经介绍过。

**再强调一次核心思想**：Spearman 只关心排序，不关心具体数值。

举个具体例子：

| 区域 | 真实违规率 | 模型 A 预测 | 模型 B 预测 |
|------|-----------|------------|------------|
| 甲   | 0.80      | 0.75       | 0.95       |
| 乙   | 0.50      | 0.48       | 0.60       |
| 丙   | 0.20      | 0.22       | 0.30       |

- **模型 A**：预测值很接近真实值（误差小），排序也正确（甲>乙>丙）
- **模型 B**：预测值与真实值差距较大，但排序也是正确的（甲>乙>丙）

在 Spearman 评分下，**模型 A 和模型 B 得分一样**！因为它们的排序都是完美的。

这意味着：
- 模型不需要精确预测 0.80，只需要知道"甲比乙高，乙比丙高"
- 这对我们的建模策略有影响——**不需要过度追求数值精度，而要追求排序的准确性**

**Python 中如何计算**：

```python
from scipy.stats import spearmanr

# 真实值和预测值
y_true = [0.80, 0.50, 0.20, 0.65, 0.35]
y_pred = [0.75, 0.48, 0.22, 0.60, 0.30]

correlation, p_value = spearmanr(y_true, y_pred)
print(f"Spearman 相关系数: {correlation:.4f}")
# p_value 越小，说明这个相关性越不是偶然的
```

### 1.3 辅助指标

虽然 Spearman 是主指标，但以下辅助指标能帮助我们更全面地理解模型表现：

| 指标 | 英文全称 | 含义 | 用途 |
|------|---------|------|------|
| MAE | Mean Absolute Error | 预测值与真实值的平均绝对误差 | 直觉上"平均偏了多少" |
| MSE | Mean Squared Error | 预测值与真实值差的平方的平均 | 对大误差惩罚更重 |
| RMSE | Root Mean Squared Error | MSE 开根号 | 和原始数据同单位，更直观 |
| R² | R-squared | 模型解释了多少比例的目标变量变异 | 1 = 完美，0 = 和取平均一样差 |

**用辅助指标的好处**：
- 如果 Spearman 高但 MAE 也很大，说明排序对了但数值偏差大——这在我们的比赛中没问题，但在实际应用中可能需要关注
- 如果 MAE 突然变大，可能是模型在某些区域预测得非常离谱——值得分析原因

### 1.4 如何分析模型表现

不要只看一个总分数！要**分组分析**，找出模型的"弱点"：

**按 total_count 分组看**：
- 模型在 total_count=1 的样本上表现如何？（这些样本噪声最大）
- 在 total_count≥30 的样本上呢？（这些数据最可靠）
- 如果模型在小 total_count 上的 Spearman 很差，这是正常的——因为 total_count=1 时目标值只能是 0 或 1，很难排序

**按区域分组看**：
- 哪些区域预测得好？哪些区域预测得差？
- 预测得差的区域有什么共性？（可能是数据太少、或者违规模式不稳定）

**按时间分组看**：
- 某些小时/月份的预测是否特别差？
- 如果是，可能需要更强的时间特征

## 二、特征重要性分析 — 模型在"看"什么？

### 2.1 为什么要分析特征重要性

训练完模型后，我们想知道：**模型到底依赖哪些特征做出预测？**

这个问题很重要，原因有三：
1. **理解模型**：知道模型的"决策逻辑"，增加对结果的信心
2. **指导特征工程**：重要性低的特征可以去掉，重要性高的方向可以继续深挖
3. **写报告和做展示**：特征重要性图是报告和 PPT 中最有说服力的图表之一

### 2.2 LightGBM/XGBoost 内置的特征重要性

这两个模型自带特征重要性计算，有两种常见的衡量方式：

**Gain（增益）**：
- 含义：这个特征在所有树的分裂中带来了多少总增益（误差的减少量）
- 比喻：这个"问题"每次被问到时，帮助我们排除了多少不确定性

**Split count（分裂次数）**：
- 含义：这个特征被用来分裂了多少次
- 注意：使用次数多不一定意味着重要——可能只是在做微调

**推荐用 Gain**，因为它更能反映特征的实际贡献。

```python
import lightgbm as lgb
import matplotlib.pyplot as plt

# 训练好模型后
lgb.plot_importance(model, importance_type='gain', max_num_features=20)
plt.title('特征重要性（按增益排序）')
plt.tight_layout()
plt.savefig('feature_importance.png')
```

### 2.3 SHAP 分析 — 更深入的解释

SHAP（SHapley Additive exPlanations，沙普利值）是一种更高级的特征解释方法。

**直觉理解**：
- 想象一群人合作完成了一个项目，怎么公平地分配功劳？
- SHAP 源自博弈论中的 Shapley 值，它考虑了每个特征在所有可能的特征组合中的边际贡献
- 比简单的 Gain/Split 更公平、更准确

**SHAP 能告诉我们什么**：
- 每个特征对每一条预测的贡献（正/负方向）
- 特征值的大小如何影响预测（例如：温度越高 → 违规率越高还是越低？）
- 特征之间的交互效应（例如：只有在 A 区域下雨时违规率才下降）

**关键的 SHAP 图**：

1. **Summary Plot（概要图）**：横轴是 SHAP 值（对预测的影响大小），每个点是一条数据，颜色代表特征值高低
2. **Dependence Plot（依赖图）**：展示某个特征值和其 SHAP 值的关系，可以加入交互特征着色

```python
import shap

# 创建解释器（对 TreeModel 非常快）
explainer = shap.TreeExplainer(model)

# 注意：607 万行全部算 SHAP 太慢，采样一部分即可
sample = X_train.sample(10000, random_state=42)
shap_values = explainer.shap_values(sample)

# Summary plot
shap.summary_plot(shap_values, sample)

# 单个特征的依赖图
shap.dependence_plot("total_count", shap_values, sample)
```

**论文中的 SHAP 应用**：
- Paper 5（Sui 2025）使用 SHAP 分析发现天气与时间存在非线性交互——这意味着简单地加入天气特征可能不够，需要创建交叉特征
- Paper 1 的消融实验（逐一去掉特征类别）发现时间特征最重要（贡献 10.2%），天气次之（4.9%），节假日（1.7%）

## 三、模型集成 — 三个臭皮匠顶个诸葛亮

### 3.1 什么是模型集成（Ensemble）

**核心思想**：把多个模型的预测结合起来，通常比任何单个模型都好。

**为什么有效？** 不同模型有不同的"偏见"和"盲点"：
- LightGBM 可能在某些区域预测得好，但在另一些区域不行
- XGBoost 的弱点可能恰好是 LightGBM 的强项
- 把它们的预测平均起来，偏差往往会互相抵消

这就像看病时想多找几个医生的意见——每个医生都有自己的专长和局限，综合多个意见通常比只听一个人的更可靠。

### 3.2 加权平均（最简单有效的集成方法）

```python
# 假设有三个模型的预测
pred_lgb = model_lgb.predict(X_test)    # LightGBM 的预测
pred_xgb = model_xgb.predict(X_test)    # XGBoost 的预测
pred_cat = model_cat.predict(X_test)    # CatBoost 的预测

# 简单平均
pred_avg = (pred_lgb + pred_xgb + pred_cat) / 3

# 加权平均（根据各模型的验证分数分配权重）
# 假设 LightGBM 最好（Spearman=0.50），XGBoost 次之（0.48），CatBoost 最差（0.45）
pred_weighted = 0.4 * pred_lgb + 0.35 * pred_xgb + 0.25 * pred_cat
```

**如何确定权重？**
- 方法一：根据交叉验证的 Spearman 分数按比例分配
- 方法二：在验证集上用网格搜索找最优权重
- 经验法则：表现好的模型给更高的权重，但不要把权重全压在一个模型上

### 3.3 Stacking（更高级的集成方法）

**思想**：训练一个"元模型"来学习如何最好地组合各个基础模型的预测。

**步骤**：
1. **第一层**：用 K-Fold 交叉验证训练多个基础模型（LightGBM、XGBoost、CatBoost 等），得到每个模型在验证集上的预测
2. **第二层**：把第一层各模型的预测作为新特征，训练一个简单的元模型（通常是线性回归或 Ridge 回归）

```
                    训练数据
                   /   |    \
            LightGBM  XGBoost  CatBoost    ← 第一层（基础模型）
                 \     |      /
           各模型的预测值（作为新特征）
                       |
                  Ridge 回归              ← 第二层（元模型）
                       |
                   最终预测
```

**为什么第二层用简单模型？** 因为第二层的"特征"只有几个（各模型的预测值），用复杂模型容易过拟合。

**注意防泄露**：第一层的预测必须来自交叉验证的 out-of-fold（OOF）预测，不能用训练数据直接预测。否则就是"自己给自己出题再自己答"，会导致过拟合。

### 3.4 集成的实际建议

| 策略 | 适用场景 | 复杂度 | 预期提升 |
|------|---------|--------|---------|
| 简单平均 | 各模型差不多好 | 低 | 小幅提升 |
| 加权平均 | 各模型表现有差异 | 低 | 中等提升 |
| Stacking | 追求最高分数 | 高 | 最大提升 |

**推荐路线**：先做好单个 LightGBM 模型 → 再加 XGBoost → 尝试加权平均 → 有时间再尝试 Stacking。

## 四、常见陷阱与避坑

### 4.1 数据泄露（Data Leakage）— 最严重的问题

**什么是数据泄露**：
模型在训练时"偷看"了它不应该看到的信息（通常来自测试集或未来的数据）。

**比喻**：就像考试前偷看了试卷——你以为自己很厉害，但其实只是在作弊，换一套题就不行了。

**在我们项目中容易发生的泄露**：

1. **Target Encoding 泄露**：
   - 错误做法：用所有训练数据（包括当前样本）计算区域平均违规率
   - 正确做法：K-Fold Target Encoding（在 Part 2 中已详细讲解）

2. **全局统计泄露**：
   - 错误做法：用包含测试集的数据计算统计特征
   - 正确做法：所有统计特征只在训练集上计算，然后映射到测试集

3. **时间泄露**（如果有时间序列的结构）：
   - 如果数据有时间顺序，不能用未来的数据来预测过去

**如何检测泄露**：如果模型在交叉验证上的分数"好得不正常"（比如 Spearman > 0.8），很可能发生了泄露。

### 4.2 过拟合（Overfitting）

**表现**：训练集上分数很高，但验证集上分数差很多。

**解决方法**：
- 使用早停（Early Stopping）— 最重要的手段
- 减小 `num_leaves` 或 `max_depth`
- 增大 `min_child_samples`
- 使用 `feature_fraction` 和 `bagging_fraction`
- 增大正则化参数（`reg_alpha`、`reg_lambda`）

### 4.3 小样本噪声

我们数据中 total_count=1 的样本有约 12.6 万条，这些样本的违规率只能是 0 或 1，噪声极大。

**应对策略**：
- 在特征工程中创建 `log_total_count` 或 `total_count` 分箱特征，帮助模型区分大/小样本
- Paper 1 使用高斯平滑（Gaussian smoothing，σ=210分钟）来平滑目标值——这是一个高级技巧，把 σ=210 分钟内的多次观测平均起来，使目标值更平滑、噪声更小
- 考虑对小 total_count 的样本降低权重（在 LightGBM 中通过 `weight` 参数实现）

### 4.4 特征选择

**不是所有特征都越多越好**：
- 太多无用特征 → 模型被干扰，性能下降
- 方法：先训练一个模型，看特征重要性，去掉重要性为 0 或接近 0 的特征
- 或者用 SHAP 值来判断

### 5.1 生成预测文件

ChallengeData 要求提交的文件格式通常是 CSV：

```python
import pandas as pd

# 用最佳模型（或集成模型）预测测试集
test_predictions = best_model.predict(X_test_features)

# 创建提交文件
submission = pd.DataFrame({
    'ID': test_ids,              # 测试集的 ID 列
    'invalid_ratio': test_predictions  # 预测值
})
submission.to_csv('submission.csv', index=False)
```

### 5.2 提交建议

- **多次提交**：不要只提交一次。每次改进模型后都提交，看看真实排行榜上的分数变化
- **不要过度依赖排行榜**：排行榜只显示测试集上的分数，如果你只针对排行榜调参（而不是交叉验证），就像只针对一套试卷复习——可能会过拟合
- **记录每次提交**：模型版本、参数、CV 分数、排行榜分数都记录下来
